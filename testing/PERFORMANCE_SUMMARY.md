# Performance Testing Summary - v10.1

Performance testing was conducted using a dual-method approach on a production-grade Hetzner deployment (16 vCPU AMD EPYC-Genoa, 30GB RAM) running Hyperledger Fabric v2.5.12 with a dual-channel architecture (electricity-de and hydrogen-de). The network topology comprised six peers across six organizations, four Raft orderers, and six CouchDB state databases.

**Read Operations** were benchmarked using Hyperledger Caliper v0.6.0 with 4 concurrent worker processes and the Fabric Gateway SDK v1.5.0 peer-gateway connector. Query functions (admin:GetVersion, query:GetCurrentEGOsList, query:GetCurrentHGOsList) achieved 40-50 transactions per second with average latencies of 10ms and maximum latencies below 40ms. All tested functions demonstrated 100% success rates across 500-transaction test rounds on both channels, validating the multi-channel deployment's operational readiness.

**Write Operations** required peer CLI-based testing due to critical limitations in Caliper's peer-gateway connector, which failed to pass transient data parameters and ignored invokerIdentity specifications—both essential for role-based access control and private data operations. Single-threaded peer CLI benchmarks of oracle:PublishOracleData achieved 8.0 TPS with 115ms average latency and 100% reliability across 50 transactions. Multi-threaded testing with 5 concurrent workers demonstrated substantial throughput improvements: oracle:PublishOracleData reached 41.9 TPS (5x improvement), backlog:AddToBacklogElectricity achieved 31.0 TPS, and issuance:CreateElectricityGO—requiring state-based endorsement from all three channel members—sustained 4.2 TPS with 133ms average latency. Success rates ranged from 63-100% under concurrent load, with failures attributable to endorsement collection timeouts and read-set validation conflicts during parallel execution rather than architectural limitations.

The performance differential between read (40-50 TPS) and write operations (8-42 TPS depending on threading and endorsement complexity) reflects the inherent consensus overhead of Fabric's endorsement, ordering, and ledger commit pipeline. Multi-threading demonstrated near-linear scaling for simple write operations, with throughput approaching Caliper's read-operation performance when using 5 concurrent workers. Cross-organizational state-based endorsement policies introduce measurable coordination overhead but maintain functional correctness, confirming the architecture's suitability for production Guarantee of Origin issuance workloads with consortium-based validation requirements.
